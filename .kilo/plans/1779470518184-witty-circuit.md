# Steel Stealth Browser Integration Plan

## Overview
Integrate Steel's stealth browser capabilities with automatic CAPTCHA solving into claude-free extension, enabling AI agents to navigate CAPTCHA-heavy sites transparently.

## Current Architecture Analysis

### Existing Components
- `src/background.ts` - Chrome extension service worker with CDP computer use (lines 177-399)
- `src/lib/computer-use.ts` - Computer tool interface (lines 25-38)
- `src/sidepanel/store.ts` - Zustand store for conversation state (lines 233-731)
- `src/lib/openai-compat.ts` - OpenAI compatibility layer (existing)

### Key Integration Points
1. Computer tool execution (`handleComputerUse` in background.ts:177)
2. Session state management (store.ts:233)
3. Provider configuration (types.ts:87-118)

## API Endpoints to Use

### Steel Sessions API
```
POST /sessions - Create session with { solveCaptcha: true }
GET /sessions/{id} - Get session status
DELETE /sessions/{id} - Close session
```

### Steel CAPTCHA API
```
GET /sessions/{id}/captchas - List detected CAPTCHAs
POST /sessions/{id}/captchas/solve - Solve CAPTCHAs
```

## Risk Mitigation

1. **Fallback Strategy**: If Steel fails, fall back to CDP browser
2. **Timeout Handling**: 30s timeout for session creation, 60s for navigation
3. **Rate Limiting**: Respect Steel's rate limits with backoff
4. **Credential Security**: API key stored in Chrome's secure storage
